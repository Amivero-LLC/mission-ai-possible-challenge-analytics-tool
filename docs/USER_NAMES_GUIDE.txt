HOW TO ADD REAL NAMES:
================================================================================

STEP 1: Open data/user_names.json in a text editor

STEP 2: Replace the default names with real names:

   BEFORE:
   "115d1995-6d11-4fb8-934f-95fe4c58d37e": "User 1"

   AFTER:
   "115d1995-6d11-4fb8-934f-95fe4c58d37e": "John Smith"

STEP 3: Save the file

STEP 4: Run the analyzer again:
   python analyze_missions.py

STEP 5: Dashboard now shows "John Smith" instead of "User 1"!

TIPS:
================================================================================

1. IDENTIFYING USERS:
   - Look at chat titles to guess who it is
   - Check which user has the most chats
   - Ask your OpenWebUI admin for user mappings

2. PARTIAL MAPPING:
   - You don't need to map all users
   - Unmapped users still show as "User X"
   - Map only the users you care about

3. KEEPING IT UPDATED:
   - When new users join, add them to data/user_names.json
   - The file won't overwrite itself
   - You maintain the mappings manually

4. PRIVACY:
   - data/user_names.json is local to your machine
   - Not included in exports
   - Safe to use real names or keep generic

TROUBLESHOOTING:
================================================================================

Q: I edited data/user_names.json but names don't change
A: Make sure:
   - JSON syntax is correct (no trailing commas)
   - You saved the file
   - You ran python analyze_missions.py again

Q: How do I find out who a user ID belongs to?
A: Options:
   - Ask your OpenWebUI administrator
   - Check the chat titles for clues
   - Export user data from OpenWebUI admin panel

Q: Can I use special characters in names?
A: Yes! But watch out for:
   - Quotes inside names: Use \"John \"Johnny\" Smith\"
   - Emojis work fine: "John Smith ðŸŽ¯"

Q: What if I delete data/user_names.json?

   - Still work perfectly
   - You can recreate it anytime
